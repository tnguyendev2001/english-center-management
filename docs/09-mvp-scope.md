# 09 - MVP Scope

## Included in MVP

Core modules:
- Dashboard
- Students
- Classrooms
- Tuition Packages
- Class Packages
- Enrollment
- Student Packages
- Class Sessions
- Attendance
- Makeup Credits
- Invoices
- Payments
- Debt
- Revenue
- Statistics
- Reports
- Activity Logs

## Main MVP flows

1. Create tuition package.
2. Create classroom.
3. Assign multiple packages to classroom.
4. Create student.
5. Enroll student into classroom and choose package.
6. Auto-create first invoice.
7. Record payment.
8. Cancel wrong payment.
9. View debt.
10. View revenue.
11. Generate class sessions.
12. Mark attendance.
13. Create makeup credit for excused absence.
14. Cancel class session.
15. Change package during active package.
16. View dashboard and reports.

## Not included in MVP

Do not implement:
- Multiple roles.
- Parent account.
- Student account.
- Mobile app.
- CRM.
- Marketing automation.
- Teacher salary.
- Multi-branch management.
- Student wallet.
- Automatic refund.
- Complex accounting.
- Microservices.
- API Gateway.
- RabbitMQ.
- Kafka.
- Event-driven architecture.
- Complex notification system.
- Auto-create next invoice without confirmation.
- Auto-change package.
- Auto-refund money.

## MVP quality requirement

The MVP must be:
- Simple to operate.
- Accurate for money.
- Accurate for session counting.
- Easy to audit.
- Easy to continue implementing with Cursor.
